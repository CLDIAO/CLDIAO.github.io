使用结构化命令

在一些简单脚本中，shell按照命令在脚本中出现的顺序依次进行处理。但许多程序要求对shell脚本中的命令施加一些逻辑流程控制。有一类命令会根据条件使脚本跳过某些命令。这样的命令通常称为结构化命令。

1. 使用if-then 语句

最基本的结构化命令就是if-then语句。格式：

if command

then

​ commands

fi

bash shell的if语句会运行if后面的那个命令。如果该命令的退出状态码是0，位于then部分的命令就会被执行。如果该命令的退出状态码是其他值，then部分的命令被跳过，bash shell会继续执行脚本中的下一个命令。fi语句用来表示if-then语句到此结束。

上面这个例子中，if语句使用了错误的命令，产生非零的退出状态码，且bash shell跳过then部分的echo语句。运行if语句中的那个错误命令产生的错误消息依然会显示在脚本的输出中。

在then部分，可以使用多条命令。bash shell会将这些命令当成一个块，如果if语句行的命令的退出状态值是0，所有的命令都会被执行；如果if语句行的命令退出状态不为0，所有的命令都会被跳过。

if语句行使用grep命令在/etc/passwd文件中查找某个用户名当前是否在系统上使用。如果用户使用了那个登录名，脚本会显示一些文本信息并列出该用户HOME目录的bash文件。但是，当testuser变量设置成一个系统上不存在的用户，则什么都不会显示。

2. if-then-else语句

如果命令返回一个非零退出状态码，bash shell会继续执行脚本中的下一条命令。这时，如果能执行另一组命令，需要if-then-else语句。

if-then-else语句在语句中提供了另外一组命令。

if command

then

​ commands

else

​ commands

fi

当if语句中的命令返回退出状态码0时，then部分中的命令会被执行。当if语句中的命令返回状态非零时，else部分中的命令会被执行。

else部分可以包含多条命令。fi语句说明else部分结束了。

3 嵌套if

有时需要检查脚本代码中的多种条件。对此，可以使用嵌套的if-then语句。

要检查/etc/psswd文件中是否存在某个用户名以及该用户的目录是否尚在，可以使用嵌套的if-then语句。嵌套的if-then语句位于主if-then-else语句的else代码块中。但在脚本中使用这种嵌套的if-then语句会使代码不易阅读，很难理清逻辑流程。

可以使用else部分的另一种形式：elif。这样不用书写多个if-then语句。

if commands

then

​ commands

elif command2

then

​ more commands

fi

elif语句行提供了另一个要测试的命令，如果elif后命令的退出状态码是0，则bash会执行第二个then语句部分的命令。使用这样的嵌套方法，代码更清晰，逻辑更易懂。

甚至可以让脚本检查拥有目录的不存在用户以及没有拥有目录的不存在用户据。这可以通过嵌套再elif中加入一个else语句来实现。

可以将多个elif语句穿起来，形成一个大的if-then-elif嵌套组合。

if command1

then

​ command set1

elif command2

then

​ command set 2

elif command 3

then

​ command set 3

elif command 4

then

​ command set 4

fi

每块命令都会根据命令是否会返回退出状态码0来执行。bash shell会一次执行if语句，只有第一个返回退出状态码0的语句中的then部分会被执行。

4 test命令

if-then语句不能测试命令退出状态码之外的条件。但bash shell中的test命令可以通过if-then语句测试不同条件的途径。如果test命令中列出的条件成立，test命令就会退出并返回退出状态码0.这样if-then语句就和其他编程语言中的if-then语句以类似的方式工作了。如果条件不成立，test命令就会退出并返回非零的退出状态码，使得if-then语句不再被执行。test命令格式：test condition。

condition是test命令要测试的一系列参数和值。当用在if-then语句中时，如下。

if test condition

then

​ commands

fi

如果不写test命令的condition部分，它会以非零的退出状态码退出，并执行else语句块。

当加入一个条件时，test命令会测试该条件。例如，确定变量中是否有内容。

bash shell有另一种测试方法，无需在if-then语句中生命test命令。

if [ condition ]

then

​ commands

fi

方括号定义测试条件。方括号和condition之间要有空格。

test命令可以判断三类条件：

  • 数值比较
  • 字符串比较
  • 文件比较

4.1 数值比较

testm命令最常用于对两个整数数值的比较（浮点数会报错）。下表为测试两个值时可用的条件参数。

比较 描述
n1 -eq n2 n1是否等于n2
n1 -ge n2 n1是否大于或等于n2
n1 -gt n2 n1是否大于n2
n1 -le n2 n1是否小于或等于n2
n1 -lt n2 n1是否小于n2
n1 -ne n2 n1是否不等于n2

数值条件测试可以用在数字和变量上。

4.2 字符串比较

条件测试允许比较字符串值，但比较麻烦。下面是常用字符串比较功能。

在比较字符串的相等性时，比较测试会将所有的标点和大小写情况都考虑在内。

当开始使用测试条件的大于或小于功能时，会出现两个问题：

  • 大于号和小于号必须转义，否则会被当做重定向符号，把字符串值当做文件名
  • 大于和小于顺序和sort命令所采用的不同

第二个问题很少遇到。sort命令处理大小写字母的方法正好和test命令相反。

4.2.3 字符串大小

-n和-z参数可以检查一个变量是否含有数据。未被定义的变量字符串长度仍然为0。

4.3 文件比较

这类是shell编程中最强大、用的最多的比较形式。可以测试Linux文件系统上文件和目录的状态。

构建基本脚本

shell脚本的关键在于输入多个命令并处理每个命令的结果，甚至将一个命令的结果传给另外一个命令。shell可以串联多个命令，一次执行完成。若两个命令一起运行，可以放在同一行，用分号隔开。

这样的小型脚本每次运行前要输入整个命令。可以将这些命令组合成一个简单的文本文件，再需要运行命令时，运行文本文件就可以。

4. 使用变量

运行shell脚本中的单个命令有自身的限制。变量允许临时将信息存储在shell脚本中，以便和脚本中的其他命令一起使用。

4.1 环境变量

脚本中也可以使用环境变量。使用时，在环境变量名称之前加上$。

echo命令中的环境变量会在脚本运行时替换成当前值。在第一个字符串中，还可以将$USER系统变量放到双引号中，shell依然能够知道我们的意图。但脚本在引号中出现$后，它会以为在引用变量。需要在$前加\符号。

4.2 用户变量

shell脚本还可以使用用户变量。定义变量允许临时存储数据在脚本中使用，从而使shell脚本看起来更像一个真正的程序。

用户变量可以是长度20以内的字母、数字、下划线组成的字符串。区分大小写。引用变量值时需要使用$，引用变量来对其进行赋值时不用使用$。

4.3 命令替换

shell脚本中最有用的特性之一就是可以从命令输出中提取信息，并将其赋给变量，之后就可以在脚本中随意使用。两种发发将命令输出赋给变量：

  • 反引号`
  • $() 格式

赋值符号和等号之间没有空格。

5. 重定向输入和输出

当想要保存某个命令的输出，bash shell提供了几个操作符，可以将命令的输出重定向到另一个位置。重定向可以用于输入，也可以用于输出，可以将文件重定向到命令输入。

5.1 输出重定向

最基本的重定向将命令的输出发送到一个文件中。bash shell用>来完成这项功能：

command > outputfile

之前显示器上出现的命令输出会被保存到指定的输出文件中。

重定向符创建了一个文件test7，并将date命令的输出重定向到该文件中。如果输出文件已经存在了，重定向操作符会用新的文件数据覆盖已有的文件。

当不想覆盖，而是将输出追加到已有文件中，可以使用>>来追加数据。

5.2 输入重定向

输入重定向和输出重定向正好相反。输入重定向将文件的内容重定向到命令，而非将命令的输出重定向到文件。

command < inputfile

wc命令可以对数据中的文本进行技术。默认情况下，它会输出三个值：

  • 文本的行数
  • 文本的词数
  • 文本的字节数

通过将文本文件重定向到wc命令，可以得到文件中的行、词和字节的计数。

另一个输入重定向的方法，成为内联输入重定向。无需使用文件进行重定向，只需要在命令行中指定用于输入重定向的数据就可以了，符号是<<。符号外，还必须指定一个文本标记来划分输入数据的开始和结尾。任何字符串都可以作为文本标记，但在数据开始和结尾文本标记必须一致。

command << marker
data
marker

在命令行上使用内联输入重定向时，shell会用PS2环境变量中定义的次提示符来提示输入数据。次提示符持续提示，知道输入作为文本标记的那个字符串。

6. 管道

有时需要一个命令的输出作为另一个命令的输入。这可以用重定向来实现，但会比较笨拙。这时我们可以使用管道连接来直接重定向到另一个命令。

command1 | command2

前一个命令执行后，随即进行第二个命令的执行。可以用一条文本分页命令来强行将输出按屏显示。

date > test8 | sort | more

date >test8 | sort > test8

more test8

7. 执行数学运算

shell脚本操作数字的过程比较麻烦。有两种途径来进行数学运算。

7.1 expr命令

这个命令特别笨拙。

7.2 使用方括号

在bash中，将一个数学运算结果赋给某个变量时，可以用$和[]（$[ operation ]）将数学表达式围起来。

这种运算比expr方便得多，也适用于shell脚本。

但bash shell的数学运算只支持整数运算。

7.3浮点解决方案

几种方法可以克服bash中数学运算的整数限制。最常见的是内建的bash计算器，即bc。

7.3.1. bc的基本用法

bash计算器实际上是一种编程语言，允许在命令行中输入浮点表达式，然后解释并计算，最后返回结果。bc能够识别：

  • 数字（整数和浮点数）
  • 变量（简单变量和数组）
  • 注释（#或/ /开始的行）
  • 表达式
  • 编程语言
  • 函数

在shell提示符下bc命令访问bash计算器；scale变量设置小数位数；quit命令退出计算器。

7.3.2 脚本中使用bcd

variable=$(echo "options; expression" | bc)

options进行设置变量，可以用分号分隔多个变量。expression定义数学表达式。

8. 退出脚本

shell中运行的每个命令都使用退出状态码告诉shell它运行完毕。退出状态码是一个0~255的整数值，在命令结束时传给shell。可以捕获这个值并在脚本中使用。

8.1 查看退出状态码

退出状态码没有标准，但有些可用的参考：

状态码 描述
0 命令成功结束
1 一般性未知错误
2 不适合的shell命令
126 命令不可执行
127 没有找到命令
128 无效的退出参数
128+x 与Linux信号x相关的严重错误
130 通过CTRL+C终止的命令
255 正常范围之外的退出状态码

8.2 exit命令

默认情况下，shell脚本以脚本中的最后一个命令的退出状态码退出。但可以改变这种默认行为，返回自己的退出状态码。exit命令可以在脚本结束时指定一个退出状态码。

exit命令的参数中也可以使用变量。
