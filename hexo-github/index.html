hexo+github

Author: Rumble Date: Oct 8, 2017 Updated On: Oct 17, 2017
Categories:
Tags: blog,hexo,github
Get Updated for Each Release!

　　近期生信技能树公众号推送了一系列我为什么写博客的推文。想到从前并未接触过这个东西，但写博客的好处大家却有共同的见解，就决定去了解一下。在微信群里看到有人说hexo+github的架构，所以花些时间来尝试一下。

　　首先放上我的链接：cldiao’s blog，10.7下午开始搞，细节的东西在之后慢慢优化吧。

    个人域名

    　　域名就是网站的入口，往往需要个人特色，现在域名也都便宜，许多人会选择购买常见后缀的域名，也有小众的免费域名供大家选择。这里因为开始入门，就先不考虑鲜明特色了，我就使用了github提供的仓库进行尝试。有需要的可参考下面参考资料或者自行谷歌。

    GitHub创建个人仓库

    　　登陆GitHub-New repository-“Github用户名.github.io”。
    　　
    　　

    安装Git

    　　网站在本地建好以后需要使用Git同步到Github上。 下载：选择合适的终端Git下载 ，细节学习：廖雪峰老师的Git教程。下载后安装完成后，找到应用程序Git Bash，打开出现窗口，说明安装成功。
    　　
    　　安装完成后，还需要进行设置用户信息，在命令行输入：
    git config --global user.name "your name"
    git config --global user.email "your email"
    生成ssh密钥文件：
    ssh-keygen -t rsa -C"your email"
    　　点击三次回车，默认不需要密码。然后找到生成的.ssh的文件夹中的id.rsa.pub的文件，打开将内容复制。
    　　
    　　打开https://github.com/settings/keys ,点击New SSH Key，Title 可随意命名，将内容粘贴到Key栏中，点击Add SHH Key。在Git Bash 中检测是否设置成功，输入ssh git@github.com
    　　
    如上则说明成功。

    安装Node.js

    　　Hexo 基于Node.js，Node.js下载地址，下载安装包。Node.js会包含环境变量及npm的安装。安装后检测Node.js是否安装成功，命令行输入
    node -v
    　　
    检测npm是否安装成功，命令行输入：npm -v
    　　
    这样Hexo的安装环境就全部搭建完成。

    安装Hexo

    　　 Hexo 是我们个人博客的框架，需要在自己的电脑上创建一个文件夹，可命名为Blog，Hexo框架与以后发布的网页都会存储在这个文件夹里。创建好文件夹后进入文件夹，按住shift键右击鼠标点击“在此处打开powershell窗口”，
    npm install -g hexo-cli#安装hexo
    hexo init#初始化hexo
    npm install#安装依赖包
    npm install hexo-server --save#Hexo3.0后server被单独出来需要安装
    hexo server#本地查看
    　　
    在浏览器中输入localhost:4000可以预览网页。有一个问题是localhost:4000会拒绝访问，可以参考下方参考链接，调整电脑设置。
    hexo g#生成网页
    hexo d#部署网页

    推送网站

    　　前面完成了本地网站的预览，接下来是发布网站，让更多的人访问。这时在blog根目录中的_config.yml称为站点配置文件 ，在theme文件夹中也有一个_config.yml被称为主题配置文件 。将Hexo和Github关联起来，需要编辑站点配置文件，翻到最下方编辑如下：
    　　
    repo后填入Github创建仓库的链接，后面加.git
    　　保存站点配置文件。输入命令npm install hexo-deployer-git --save安装Git部署插件。
    　　输入三条命令：hexo clean
    hexo g
    hexo d
    　　这时在浏览器输入放置个人网站的仓库路径，就会发现博客已经上线，可以被访问了。

    图床

    　　之前在github上编辑过一些文字，发现可以创建文件夹将图片上传，就是需要自己进行分类，没办法批量预览，当然比较隐私的重要的东西就不要上传了，我好像没看到删除的选项。当然大家也会选择七牛什么的网盘。大家各取所爱吧。对了，在github里存储图片的话，需要在仓库之外存储，如新建一个repo，不然网页部署后仓库会更新删除异常文件。

    发布文章

    　　当想要发布博客时，在命令行输入hexo n "博客名字",这时会发现blog目录下source文件夹里_post文件夹里多了一个博客名字.md的文件，使用markdown编辑器可以打开，这样就可以开始我的博客之旅了！markdown编辑语法简单，可简单搜多一下。编辑完成保存后，在命令行输入hexo server进行预览，hexo g生成网页，hexo d部署网页，之后可以在浏览器查看博客更新状况。

    小结

    　　嗯，7号下午开始搞，到晚上发现写新博客预览时总是报错换行字符的问题，因为昨天是用的typora这个编辑器，而markdown编辑对文本格式稍有要求，如全角半角的。我之前一直使用微软输入法，并没有这个切换，所以下载了个搜狗输入法，并在notepad中进行编辑。这样就是需要在网页预览时才能看到总体效果。今天预览效果也还不错。先上传吧。有错误的地方请大家批评指正。
